# CSI606-2025-01---Remoto---Proposta-Trabalho-Final

# Resumo
Será um sistema de gestão especializado para vendedores e lapidadores de pedras preciosas, focado em rastrear o ciclo de vida completo da gema, desde a aquisição bruta até a venda final, integrando estoque, processo de produção (lapidação) e controle financeiro.

# Funcionalidades Implementadas

### 1. Módulo de Inventário (Estoque)
Cadastrar Pedras Brutas: Registrar a entrada de material (ex: lote de esmeraldas brutas), incluindo peso (em gramas ou quilates), custo de aquisição,  data.
Cadastrar Lote: Faz o resgistro de um lote e vincula pedras a ele
Gerenciar Estoque: Ter uma visão clara de quantas pedras (brutas) estão disponíveis.

### 2. Módulo Financeiro
Contas a Pagar: Lançar custos de aquisição (compra de brutas) 
Relatório de Lucratividade:(Custo da Pedra Brutas).

### 3. Gerenciamento Básico
Cadastro de Usuarios e Empresas.
Um usuario so é cadastrado mediante o registramento de uma empresa no sistema

# Principais desafios e dificuldades
### 1. Interação do backend com frontend
Devido ao fato de ter feito o backend em ASP.NET core e o frontend, em react ao invés de utilizar o formato mvc do ASP.NET core com as paginas Razor, foi necessário criar interfaces de tipo para o typescript no front. Com isso, uma atenção maior foi demandada ao consultar a API desenvolvida.

# Instruções para instalação e execução
* No Visual Studio abrir o arquivo .slnx
* A partir do Docker-Compose, subir o banco de dados
* Executar o backend no Visual Studio 
* Executar o frontend apartir do npm run dev
